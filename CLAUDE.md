# CLAUDE.md — rmc_glim

## Project Overview

Standalone ROS2 Jazzy workspace for **GLIM 3D SLAM** on the RMC 2.0 robot.
GitHub: `AbdullahGM1/rmc_glim`

Goal: run GLIM against recorded bag data to produce a 3D map of the environment.

## ROS1 → ROS2 Bag Conversion

New bag recordings come as ROS1 `.bag` files and must be converted before use. The conversion tooling lives in `ros2_bags/ros1_bag_convert/`.

```bash
cd ros2_bags/ros1_bag_convert

# One-time setup — creates venv/ and installs rosbags
./setup.sh

# Convert a bag — output lands in ros2_bags/<bag_name>/
./convert_bag.sh /path/to/your_file.bag
```

The `venv/` directory is gitignored — run `setup.sh` once per machine.

## Available Bags

| Bag | Path | Duration | Message count | Has /tf_static | Notes |
|---|---|---|---|---|---|
| lower_fused | `ros2_bags/lower_fused/` | ~293 s | 85,124 | Yes | **Active — primary bag** |
| fused_upper | `ros2_bags/fused_upper/` | ~112 s | 32,720 | Yes | Upper LiDAR only |
| merged_bag | `ros2_bags/merged_bag/` | ~225 s | 65,722 | Yes | ROS1-era timestamps — use with caution |

**Do not loop bags** (`-l` flag) — GLIM crashes when sim time jumps backwards on loop restart.

### Playing a bag

```bash
# From inside ros2_bags/
cd ros2_rmc_ws/src/rmc_glim/ros2_bags
ros2 bag play lower_fused --qos-profile-overrides-path qos_override.yaml --clock
```

The `qos_override.yaml` forces `/tf_static` to `transient_local` reliability so GLIM receives it correctly.

### Switching bags in the launch file

The active bag is set at the top of `glim.launch.py` as commented options. Uncomment one, comment the rest:

```python
# BAG_PATH = '.../ros2_bags/merged_bag'
# BAG_PATH = '.../ros2_bags/fused_upper'
BAG_PATH = '.../ros2_bags/lower_fused'   # ← active
```

`BAG_NAME` is derived automatically from `BAG_PATH` for the map output directory name.

## Bag Topics

| Topic | Type | Count (lower_fused) | Notes |
|---|---|---|---|
| `/velodyne_points` | `sensor_msgs/PointCloud2` | 5,827 | Primary LiDAR input to GLIM |
| `/imu` | `sensor_msgs/Imu` | 29,387 | **Gravity-subtracted** — do NOT use for GLIM |
| `/imu/gravity` | `sensor_msgs/Imu` | 29,388 | **Raw gravity vector** — use this for GLIM |
| `/tf_static` | `tf2_msgs/TFMessage` | 1 | Full arm joint chain + `chassis_link → imu_link` |
| `/tf` | `tf2_msgs/TFMessage` | 7,346 | Dynamic arm/flipper joints only — no `map → odom` |
| `/odom/raw` | `nav_msgs/Odometry` | 7,348 | Wheel odometry — not used by GLIM |
| `/scan` | `sensor_msgs/LaserScan` | 5,827 | 2D scan — not used by GLIM |

### IMU Topics — Critical Detail

```
/imu         → linear_acceleration ≈ [0.06, 0.0, -0.07] m/s²  ← gravity subtracted (≈0)
/imu/gravity → linear_acceleration ≈ [-0.16, 0.69, 9.71] m/s² ← raw gravity vector
```

**Always use `/imu/gravity`** as `imu_topic` in `config_ros.json`. Using `/imu` causes GLIM to produce a randomly-oriented or flipped map because gravity is already removed and GLIM cannot determine the "up" direction.

## TF Frame Hierarchy

```
map
 └── odom              (published by glim_rosnode — updated with loop closure)
      └── base_link    (published by glim_rosnode — GPU VGICP odometry)
           └── velodyne  (from bag's /tf_static — arm joint chain)
```

`/tf_static` from the bag provides:
```
base_link → chassis_link → arm_flange_link → ... → velodyne_base_link → velodyne
chassis_link → imu_link   (translation: x=0.424, y=0.007, z=-0.032 — no rotation)
```

No static transform override is needed in the launch file. Adding one causes a TF conflict that breaks visualization.

## GLIM Prerequisites

GLIM is not in the standard ROS apt repository. Install from koide3's PPA:

```bash
# 1. Add PPA
sudo apt install curl gpg
curl -s https://koide3.github.io/ppa/setup_ppa.sh | sudo bash

# 2. Install GLIM (match your CUDA version — check with nvidia-smi)
sudo apt install -y libiridescence-dev libboost-all-dev libglfw3-dev libmetis-dev
sudo apt install -y libgtsam-points-cuda13.1-dev   # or cuda12.6, cuda12.2
sudo apt install -y ros-jazzy-glim-ros-cuda13.1    # or matching variant
sudo ldconfig

# 3. Fix missing libcudart.so.13 (if not installed system-wide)
sudo ln -s /usr/local/lib/ollama/cuda_v13/libcudart.so.13 /usr/local/lib/libcudart.so.13
sudo ldconfig
```

**GTSAM conflict:** ROS Jazzy ships GTSAM 4.2.0; koide3's PPA installs GTSAM 4.3.0. The launch file must prepend `/usr/local/lib` to `LD_LIBRARY_PATH` to force the correct version.

## Build and Run

```bash
cd ros2_rmc_ws
colcon build --packages-select slam_glim --symlink-install
source install/setup.bash
ros2 launch slam_glim glim.launch.py
```

`--symlink-install` is required — the launch file uses `os.path.realpath(__file__)` to locate `src/` for map saving. Without it, the path resolves to `install/` instead.

GLIM's built-in **Iridescence 3D viewer** opens automatically. No RViz needed.

### What the launch file does

| Time | Event |
|---|---|
| t=0s | Bag starts playing with `--clock` and QoS overrides, `/tf_static` published |
| t=3s | `glim_rosnode` starts — gravity initialized from first `/imu/gravity` reading |
| t=3s+ | Iridescence viewer opens, 3D map builds in real time with loop closure |
| bag end | 5-second grace period for GLIM to finish remaining scans |
| bag end +5s | `Shutdown()` → GLIM saves map dump to `maps/<bag_name>_<timestamp>/` |

## Map Saving

Maps are saved automatically when the bag finishes. Each run creates a **timestamped subdirectory** so previous maps are never overwritten:

```
maps/<bag_name>_<YYYYMMDD_HHMMSS>/
```

Example: `maps/lower_fused_20260512_143021/`

### Saved map contents

```
maps/lower_fused_<timestamp>/
├── 000000/, 000001/, ...  # Individual submaps (point clouds + poses)
├── graph.bin / graph.txt  # Pose graph with loop closure factors
├── odom_lidar.txt         # Per-frame LiDAR odometry trajectory
├── odom_imu.txt           # Per-frame IMU odometry trajectory
├── traj_lidar.txt         # Full optimized LiDAR trajectory
├── traj_imu.txt           # Full optimized IMU trajectory
├── values.bin             # Optimized pose graph values
└── config/                # Config snapshot used for this run
```

### Viewing saved maps

```bash
# Offline viewer (reload and inspect saved map)
LD_LIBRARY_PATH=/usr/local/lib:$LD_LIBRARY_PATH \
  ros2 run glim_ros offline_viewer --ros-args -p dump_path:=maps/lower_fused_<timestamp>

# Map editor
LD_LIBRARY_PATH=/usr/local/lib:$LD_LIBRARY_PATH \
  ros2 run glim_ros map_editor --ros-args -p dump_path:=maps/lower_fused_<timestamp>
```

## Map Orientation — T_lidar_imu

The gravity vector from `/imu/gravity` is `[-0.16, 0.69, 9.71]` — not perfectly vertical. The 0.69 y-component causes a ~4.17° tilt. `T_lidar_imu` in `config_sensors.json` applies a pre-rotation to cancel this:

```json
"T_lidar_imu": [0.0, 0.0, 0.0, 0.035460, 0.008223, 0.0, 0.999337]
```

Format: `[tx, ty, tz, qx, qy, qz, qw]`. This is a 4.17° rotation around `[0.974, 0.225, 0]`.

If a new bag is recorded with a different robot pose, recompute with:

```python
import math
g = [-0.16, 0.69, 9.71]   # from: ros2 topic echo /imu/gravity --once
mag = math.sqrt(sum(x**2 for x in g))
v1 = [x/mag for x in g]
v2 = [0.0, 0.0, 1.0]
axis = [v1[1]*v2[2]-v1[2]*v2[1], v1[2]*v2[0]-v1[0]*v2[2], v1[0]*v2[1]-v1[1]*v2[0]]
axis_mag = math.sqrt(sum(x**2 for x in axis))
axis_n = [x/axis_mag for x in axis]
theta = math.atan2(axis_mag, sum(a*b for a,b in zip(v1,v2)))
qw = math.cos(theta/2); s = math.sin(theta/2)
print([0,0,0, axis_n[0]*s, axis_n[1]*s, axis_n[2]*s, qw])
```

## Git and Large Files

### .gitignore

A `.gitignore` is in the repo root. Key entries:

| Pattern | Reason |
|---|---|
| `ros2_bags/ros1_bag_convert/venv/` | Local Python venv — not committed |
| `*.db3` / `*.bag` | Tracked via Git LFS instead |
| `maps/` | Map dumps are local output — not committed |

### Git LFS

Bag files (`.db3`) must be tracked with **Git LFS** — they are too large for regular git pushes.

```bash
# One-time setup (already done if .gitattributes exists)
git lfs install
git lfs track "*.db3" "*.bag"
git add .gitattributes

# Normal workflow
git add <files>
git commit -m "message"
git push origin main
```

Large bag files are stored in Git LFS, not in the regular git object store.
